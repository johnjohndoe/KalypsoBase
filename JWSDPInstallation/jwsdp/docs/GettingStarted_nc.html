Getting Started with JavaTM Web Services Developer Pack 1.4 with No Web Container

If you installed Java WSDP 1.4 but selected the no Web container option, you can do the following:

  • Use the following technologies, which do not require a Web container:
    • JAXB
    • JAXP
    • JAXR
    • SAAJ
    • XML Digital Signature
  • Run one of the following integration scripts to integrate your Java WSDP 1.4 installation with one of the following Web containers:
    • <JWSDP_HOME>/jwsdp-shared/bin/jwsdponsjsas.sh (Solaris/Linux) or <JWSDP_HOME>/jwsdp-shared/bin/jwsdponsjsas.bat (Windows) to integrate with Sun Java System Application Server Platform Edition 8
    • <JWSDP_HOME>jwsdp-shared/bin/jwsdponsjsws.sh (Solaris/Linux) or <JWSDP_HOME>jwsdp-shared/bin/jwsdponsjsws.bat (Windows) to integrate with Sun Java System Web Server 6.1
    • <JWSDP_HOME>jwsdp-shared/bin/jwsdpontomcat.sh (Solaris/Linux) or <JWSDP_HOME>jwsdp-shared/bin/jwsdpontomcat.bat (Windows) to integrate with the Tomcat 5.0 for Java WSDP Container

Make sure that you read the Integration Notes before you run the integration scripts. These notes describe property settings that you must modify before you can run the scripts and, if you run the integration scripts, steps you must take before you uninstall the Java WSDP.

See the index page at <JWSDP_HOME>/docs/index.html for links to the API documentation, release notes, and reference documentation for the Java WSDP technologies.
