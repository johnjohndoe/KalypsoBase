JSR 105 v0.11 API Changes

General

  1. Added new package javax.xml.crypto.dsig.dom and moved DOMSignContext and DOMValidateContext classes from javax.xml.crypto.dom package to here.
  2. Added DOM Mechanism requirement that implementations must support DOMStructures as the mechanism for applications to specify any elements or mixed content.

javax.xml.crypto

  1. Added new NoSuchMechanismException class which is an unchecked exception thrown by getInstance methods when no support for the requested mechanism is available.
  2. Added a warning to the XMLCryptoContext class that contexts may contain state and are not meant to be reusable and that each context should only be used for signing/validating one XMLSignature.
  3. Removed the NodeSetData.contains() method.

javax.xml.crypto.dom

  1. Changed back the return type of DOMURIReference.getHere() from Attr to Node.

javax.xml.crypto.dsig

  1. Rename CanonicalizationMethod constants.
  2. In getInstance methods, add hyperlink to list of standard mechanism types.
  3. Removed the supported keys from the XMLValidateContext and XMLSignContext classes.
  4. Removed Validity interface.
  5. Added a sentence to the validate methods to state that the method always returns a cached value after the first successful invocation.
  6. Removed the Manifest.validate() method.
  7. Added a SignedInfo.getKeySelectorResult() method that return the key selector result after signing/validating.
  8. Removed Reference.Validity and Reference.CachedData classes and moved methods into the Reference class.
  9. Added Reference.getCalculatedDigest() and SignedInfo.getCalculatedSignatureValue methods.
  10. In XMLObject class, content should be returned as instances of public XMLStructure subclasses, if they exist.

javax.xml.crypto.dsig.spec

  1. Renamed HMACParameterSpec.getHMACOutputLength() method to getOutputLength.
  2. Renamed XPath class to XPathType.
